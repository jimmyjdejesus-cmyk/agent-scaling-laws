# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in agent-scaling-laws, please report it responsibly.

### How to Report

**Email**: jimmyj.dejesus@gmail.com  
**Subject**: [SECURITY] agent-scaling-laws vulnerability

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### What to Expect

- **Response Time**: Within 48 hours
- **Updates**: Every 3-5 days until resolved
- **Fix Timeline**: Critical issues within 7 days
- **Credit**: You'll be credited in release notes (if desired)

### Please DO NOT

- ❌ Create public GitHub issues for security vulnerabilities
- ❌ Disclose the vulnerability publicly before a fix is available
- ❌ Exploit the vulnerability maliciously

### Our Commitment

We take security seriously and will:
- Respond promptly to reports
- Work with you to understand and fix the issue
- Keep you informed of progress
- Credit you appropriately (with your permission)
- Release security advisories for significant issues

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅ Yes    |
| 0.1.x   | ⚠️ Security fixes only |
| < 0.1   | ❌ No longer supported |

## Security Best Practices

When using agent-scaling-laws:
- Keep dependencies updated
- Use virtual environments
- Review agent code before execution
- Monitor agent behavior in production
- Set resource limits for agents
- Validate all inputs

Thank you for helping keep agent-scaling-laws secure!
